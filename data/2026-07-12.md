### [1] [DRIFTLENS: Measuring Memory-Induced Reasoning Drift in Personalized Language Models](https://arxiv.org/abs/2607.02374)
*Xi Fang,Weijie Xu,Yingqiang Ge,Yuhui Xu,Stephanie Eckman,Chandan K. Reddy*

Main category: cs.AI

TL;DR: 这篇文章提出了DRIFTLENS，用于在没有标准答案的开放式问题中衡量个性化记忆是否会改变模型的推理轨迹，并发现用户属性记忆会显著引起推理漂移。


<details>
  <summary>Details</summary>
Motivation: 现有个性化大模型通常会将用户属性、偏好和历史上下文注入提示词，但人们主要关注输出内容是否变化，而较少研究其是否会改变模型的推理过程。本研究关注：当问题没有唯一正确答案时，用户属性记忆是否会重塑模型的推理路径，以及这种影响能否被量化和缓解。

Method: 作者提出DRIFTLENS，一个无需真实标签的框架：先将模型输出中的每个显式推理步骤映射到某种价值类别，再比较“无记忆轨迹”和“注入用户属性记忆后的轨迹”之间的差异，从而度量推理漂移。论文先验证该方法能够区分无内容的语用噪声与实质性推理变化；随后在4个LLM和10类用户属性（如年龄、职业、残障等）上系统评估记忆引起的推理变化；最后还测试了基于GRPO和DPO的后训练方法对漂移的抑制效果。

Result: 实验表明，用户属性记忆在所有测试模型上都会带来高于语用噪声底线的中到大幅度推理漂移，即使最终答案仍然通顺、相关且看似合理。GRPO和DPO后训练都能降低漂移，但没有一种方法在所有模型和奖励设置下都稳定最优；同时，它们对下游能力、帮助性和指令遵循的影响也依赖于具体模型与奖励设计。

Conclusion: 个性化记忆不仅会改变模型回答什么，还会改变模型如何推理。记忆诱发的推理漂移是一种可测量、且只能部分缓解的失效模式，说明个性化语言模型在使用用户属性时需要同时关注输出内容与内部推理一致性。

Detailed Summary: 1. 这篇文章解决了什么问题：
- 研究个性化记忆是否会改变大模型的推理轨迹，而不仅仅是改变最终回答内容。
- 重点关注开放式问题，因为这类问题往往没有唯一标准答案，更容易暴露推理路径被记忆影响的现象。
- 目标是提出一种无需真实标注答案的方法，来量化“记忆诱发的推理漂移”，并探索其缓解手段。

2. 有哪些相关工作：
- 相关背景包括现代LLM中的个性化机制，即将用户属性、偏好和历史上下文存入记忆，并在后续提示中注入。
- 现有工作通常更关注个性化对输出风格、相关性或任务表现的影响，而较少研究其对推理过程本身的影响。
- 论文还涉及后训练对齐方法，如GRPO和DPO，用于比较不同训练策略是否能降低记忆带来的推理偏移。

3. 采用了什么研究方法：
- 提出DRIFTLENS框架，核心思想是把模型输出中的每一步推理映射为一个“价值类别”，然后比较有无用户属性记忆时的推理轨迹差异。
- 该方法不依赖真实答案标签，适合开放式问题分析。
- 先进行验证实验，确认DRIFTLENS可以区分无实质意义的语用噪声与真正的推理变化。
- 再在多个模型和多种用户属性条件下测量推理漂移，最后用GRPO、DPO后训练测试缓解效果。

4. 做了哪些实验、结果如何：
- 在4个LLM上测试了10类用户属性记忆，包括年龄、职业、残障等。
- 结果显示：用户属性记忆会在各模型中引发中到大幅度的推理漂移，而且这种漂移明显高于模型本身的语用噪声水平。
- 即便最终回答仍然流畅、相关、看似合理，内部推理路径也已经发生变化。
- 对GRPO和DPO的评估表明，两者都能降低漂移，但效果并不稳定：不同模型、不同奖励设置下，优劣并不一致。
- 此外，漂移抑制与下游能力、帮助性、指令遵循之间存在依赖关系，说明缓解策略具有权衡性。

5. 结论是什么：
- 个性化记忆会实质性改变LLM的推理轨迹，这是一种独立于最终答案质量的风险。
- DRIFTLENS能够在无标准答案条件下有效刻画这种变化。
- 后训练可以部分减少漂移，但尚不能彻底解决，且不同方法没有普遍最优解。

整体总结：
- 这篇论文从“个性化不仅改变输出，也改变推理”这一视角出发，提出DRIFTLENS来测量用户属性记忆导致的推理漂移。
- 通过多模型、多属性实验，作者证明了这种漂移普遍存在且具有显著强度。
- 同时，论文评估了GRPO和DPO等后训练方法，发现它们只能部分缓解问题。
- 文章的核心贡献是将“记忆影响推理过程”从直觉性担忧变成了可测量、可比较、可部分优化的研究问题。

Abstract: Personalization changes what a model says to a user; we show that it can also change the reasoning trajectory used to justify the response. Modern LLMs personalize interactions by storing user attributes, preferences, and prior context, then injecting this information into future prompts. We study whether such memory reshapes reasoning on open-ended questions where no single ground-truth answer exists. To quantify this effect, we introduce DRIFTLENS, a ground-truth-free framework that maps each expressed reasoning step to a value category and measures divergence between a question's no-memory trajectory and its trajectory under injected user-attribute memory. We first validate that DRIFTLENS distinguishes content-free pragmatic noise from substantive reasoning changes. Across four LLMs and 10 user-attribute categories, including age, occupation, and disability, user-attribute memory induces medium-to-large reasoning drift above each model's pragmatic-noise floor, even when final answers remain fluent, on-topic, and plausible. We then evaluate GRPO- and DPO-based post-training methods for reducing drift. Both reduce drift, but neither uniformly dominates; effects on downstream capability, helpfulness, and instruction following are model-and reward-dependent. These results suggest that memory-induced reasoning drift is a measurable and only partly mitigated failure mode of personalized language models.

### [2] [User as Engram: Internalizing Per-User Memory as Local Parametric Edits](https://arxiv.org/abs/2606.19172)
*Bojie Li*

Main category: cs.AI

TL;DR: 本文提出“User as Engram”分层个性化记忆方案：将用户内容记忆写入按哈希索引的Engram记忆表中，将推理能力保留在共享适配器里，以更小的存储开销获得更好的间接推理表现，并避免对无关文本的污染。


<details>
  <summary>Details</summary>
Motivation: 现有个性化方法要么把用户事实放在模型外部（如自然语言记忆文件、检索索引），要么像每用户LoRA那样把事实与推理能力一起写入权重，容易造成全局干扰。作者希望把“内容记忆”和“推理技能”分离，模拟大脑中海马体与新皮层的分工，从而实现更安全、更高效、可组合的个性化记忆。

Method: 提出User as Engram框架：1）用户事实作为对Engram模型哈希键记忆表的局部、外科式编辑写入；2）推理能力由一个共享适配器承担；3）检索时仅在触发位置激活对应记忆项，其他位置保持字面级不变；4）不同用户的事实由于落入不同哈希槽，可以在同一张共享表中叠加共存。该设计将内容与技能解耦，并保持记忆修改的局部性与可组合性。

Result: 与每用户LoRA相比，该方法在直接召回上保持竞争力，同时在间接推理任务上平均提升5.6倍；在推理能力上，从未使单个用户比原始基座模型更差。其记忆占用约减少33,000倍。与检索式流水线相比，当事实规模超过约100条后，在一个大约2.5倍更大的模型上，Engram方案开始占优。

Conclusion: User as Engram证明了将用户内容存入局部记忆表、将推理能力放入共享模块的分层设计是有效的：既能保持个性化事实的精确写入与独立组合，又能显著提升间接推理性能并降低存储和检索成本。

Detailed Summary: 1. 这篇文章解决了什么问题：
- 解决语言模型个性化记忆中“内容记忆”和“推理技能”混在一起的问题。
- 现有每用户LoRA把用户事实和通用能力一并编码进同一个权重增量，容易污染无关输出，并且不利于多用户共存。
- 作者希望实现一种像人脑那样的记忆分层：用户事实只影响局部记忆，通用推理能力保持共享且稳定。

2. 有哪些相关工作：
- 现有个性化通常采用外部记忆方案，如自然语言记忆文件或检索索引。
- 另一类方法是每用户LoRA，将用户信息写入模型权重。
- 文章对比的核心对象还包括检索式管线，以及“基座模型不加个性化”作为参照。

3. 采用了什么研究方法：
- 设计“User as Engram”分层架构。
- 将用户内容存入Engram模型的哈希键记忆表，采用局部、精确、不可扩散的写入方式。
- 将推理相关能力放在共享适配器中，避免不同用户事实相互干扰。
- 利用哈希槽分离不同用户的记忆，使多个用户的编辑可以在同一表中无损叠加。
- 通过层间分工实现“写入内容不改变其他位置”的玻璃盒式记忆编辑。

4. 做了哪些实验、结果如何：
- 评估直接召回与间接推理能力。
- 与每用户LoRA相比，该方法在间接推理上平均提升5.6倍。
- 在推理能力上，任何单个用户都不会比未个性化的基座模型更差。
- 在存储效率上，记忆占用约比LoRA小33,000倍。
- 在多用户可组合性方面，不同用户的事实可同时存在于共享表中，且是加性、无损的。
- 与检索式方法相比，当用户事实规模超过约100条时，Engram方案开始更有优势，即使是在一个更大的模型设置下也是如此。

5. 结论是什么：
- 分离“内容记忆”和“推理技能”是个性化语言模型的更优设计原则。
- 局部哈希记忆表负责精确存储用户事实，共享适配器负责通用推理，可同时获得高准确性、低干扰和高压缩率。
- 该方案尤其适合多用户、可组合、长期扩展的个性化场景。

整体总结：
- 这篇文章提出了一种新的个性化记忆范式，用Engram式局部表存用户事实、用共享适配器存推理能力，替代将两者混合进单一LoRA的做法。
- 其核心贡献是：更强的间接推理、更低的存储成本、更好的多用户共存性，以及对无关输出的最小扰动。

Abstract: Personal memory in a language model is two problems: content and reasoning skill. The brain keeps the two apart (a sparse, local engram in the hippocampus for each episode, a slow neocortex for the shared skills that interpret it), so a new fact need not overwrite everything else. Most personalization today keeps a user's facts outside the weights, in a natural-language memory file or a retrieval index. When facts are written into the model instead, the standard recipe is the per-user LoRA adapter, which does the opposite of the brain, folding content and skill into one global weight delta. Writing a user's facts as a LoRA contaminates text unrelated to them; writing the same facts as local Engram rows leaves it mathematically untouched, resulting in a roughly 33,000x smaller memory footprint.
  We therefore propose User as Engram: store a user's content as surgical edits to the hash-keyed memory table of an Engram model, and carry the reasoning skill in one shared adapter. This layered design matches per-user LoRA's direct recall while delivering 5.6x higher indirect-reasoning accuracy on average, and never makes a single user worse at reasoning than the untouched base. The edit is a glass box: writing a fact switches on its lookup at exactly the trigger, adds the value the answer needs, leaves every other position unchanged to the last bit, and fails if written into the wrong layer. Because different users' facts land in disjoint hash slots, their edits compose: many users live in one shared table at once, stacking additively and losslessly, where a per-user LoRA, a single global weight delta, admits only one. Upon retrieval, a per-user Engram table does not grow with the population the retriever must search, so past ~100 facts it overtakes a retrieval pipeline on a 2.5x larger model.

And because its rules execute deterministically whenever the state changes, UaC can surface unsolicited, safety-critical alerts -- such as a newly prescribed drug that conflicts with an allergy recorded months earlier -- a capability query-driven memory cannot provide.
